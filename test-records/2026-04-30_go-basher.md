### Issue #57: TOCTOU Race Condition
- **Location**: `basher.go:56-62`, `Application()` function
- **Problem**: `os.Stat(bashPath)` + `RestoreAsset(bashDir, "bash")` is a TOCTOU race — two concurrent first-run processes both detect the missing binary and write simultaneously
- **Impact**: One process reads a half-written ELF/Mach-O binary → crash

### Phase 2: Fix Implementation

**Strategy**: Extract to temp directory, then `os.Rename` atomically

```go
// Before (TOCTOU vulnerable):
err = RestoreAsset(bashDir, "bash")

// After (atomic):
tmpDir, _ := os.MkdirTemp(bashDir, "extract-*")
defer os.RemoveAll(tmpDir)
RestoreAsset(tmpDir, "bash")
os.Rename(tmpDir+"/bash", bashPath)
```

**Key design decisions**:
1. `os.MkdirTemp` in same filesystem as `bashPath` → `os.Rename` is atomic
2. `defer os.RemoveAll(tmpDir)` cleans up on any error path
3. If two processes race, both extract to separate temp dirs, then `os.Rename` — second rename overwrites with identical content (harmless)

## Issues Encountered

### Issue 1: go fmt changes in generated files
- **Symptom**: `go fmt` auto-formatted build-tagged bindata files, adding `//go:build` directives and fixing comment indentation
- **Root Cause**: Generated files from older `go-bindata` version don't match Go 1.22 `go fmt` expectations
- **Solution**: Reverted bindata files — kept only `basher.go` in the commit to maintain focus
- **Prevention**: Run `go fmt` only on hand-written code, not generated files

### Issue 2: Initial fix approach was wrong
- **Symptom**: First attempt used `os.CreateTemp` + `RestoreAsset(bashDir, "bash")` — temp file was created but `RestoreAsset` still wrote to `bashDir/bash` directly
- **Root Cause**: `RestoreAsset(dir, name)` always writes to `dir/name` — can't redirect output to an arbitrary file
- **Solution**: Changed to temp directory approach: `RestoreAsset(tmpDir, "bash")` + `os.Rename`
- **Prevention**: Read `RestoreAsset` signature carefully before designing the fix
